## 構築手順
```bash
$ npm create nuxt@latest my-nuxt-app -- -t v3
$ npm install
```
## ディレクトリ構成と各ファイルについて
### 主要ファイル（最初からある）
1. `app.vue` - アプリケーションのエントリーポイント
    - アプリケーション全体のルートコンポーネント
    - `pages`ディレクトリが存在しない場合、このファイルがメインページとして機能

2. `nuxt.config.ts` - Nuxtの設定ファイル
    - アプリケーション全体の設定を管理
    - モジュール、ビルド設定、ルーティングなどをここで設定可能

3. `package.json` - プロジェクト情報とスクリプト
    - スクリプト
        - `dev`: 開発サーバーを起動
        - `build`: 本番用にビルド
        - `generate`: 静的サイトとして生成（SSG）
        - `preview`: ビルド後のアプリをプレビュー
        - `postinstall`: インストール後に自動実行（型定義など準備）
    - 依存関係
        - `nuxt`: Nuxt3フレームワーク本体
        - `vue`: Vueフレームワーク
        - `vue-router`: ルーティングライブラリ

4. `tsconfig.json` - TypeScript設定
    - `tsconfig.json`を継承
    - Nuxtが自動生成する型定義を使用

5. `README.md` - プロジェクトドキュメント
    - プロジェクトの説明や使い方を記載

### 主要ディレクトリ（最初からある）
1. `/public` - 静的ファイル配置
    - ここに配置したファイルはそのままルートパスで提供される
    - `robots.txt`: 検索エンジン向けのクローラー制御ファイル
    - 画像、ファビコン、その他の静的アセットを配置

2. `/server` - サーバーサイドコード
    - APIルート、サーバーミドルウェアなどを配置
    - `tsconfig.json`: サーバー側のTypeScript設定

### 今後追加される可能性のあるディレクトリ        
- `/pages`: ページコンポーネント(ファイルベースルーティング)
- `/components`: 再利用可能なVueコンポーネント
- `/composables`: Vue Composition API の再利用可能なロジック
- `/layouts`: ページレイアウトテンプレート
- `/middleware`: ルートミドルウェア
- `/plugins`: Vueプラグイン
- `/assets`: コンパイルされる静的アセット(SCSS、画像など)
- `/stores`: Pinia ストア(状態管理)
- `.nuxt`: 自動生成される開発用ファイル(Git管理外)
